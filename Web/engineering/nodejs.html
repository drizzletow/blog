                          # Node和依赖管理

                          # 一 Node版本管理—nvm

                          # 1. 什么是nodejs

                          JavaScript是脚本语言，而脚本语言需要一个解析器才能运行。

                          在不同的地方有不一样的解析器，对于写在html中的js代码，浏览器是它的解析器。

                          而对于独立的JavaScript代码，nodejs就是一个解析器。(nodejs: http://nodejs.cn/open in new window)

                          node.js是一个基于Chrome v8引擎的javascript 运行环境。Node.js使用了一个事件驱动、非阻塞式 I/O的模型，使其轻量又高效。

                          nodejs能做什么？

                          简单可至使用命令交互模式调试JS代码片段，复杂则可编写工具提升工作效率

                          NodeJS的作者说，他创造NodeJS的目的是为了实现高性能Web服务器

                          在我们的日常开发中经常会遇到这种情况：手上有好几个项目，每个项目的需求不同，每个项目很可能依赖不同版的 NodeJS 运行环境。 如果没有一个合适的工具，这个问题将非常棘手。 —— nvm 应运而生。

                          # 2. nvm-windows

                          nvm 是 Mac 下的 node 管理工具，有点类似管理 Ruby 的 rvm， 如果需要管理 Windows 下的 node，官方推荐使用 nvmw 或 nvm-windows。（nvm-windows 并不是 nvm 的简单移植，他们也没有任何关系）

                          1. nvm-windows的安装: 首先一定要卸载已安装的 NodeJS，否则会发生冲突。

                          然后下载 nvm-windows:

                          nvm-windows：https://github.com/coreybutler/nvm-windows/releasesopen in new window （下载后直接安装即可）


                          1. 配置nvm-windows的node下载源

                          安装完成后，找到nvm文件夹下settings.txt，新增下面内容(下载源) , settings.txt文件默认所在地址： C:\Users\t\AppData\Roaming\nvm

                          arch: 64 
proxy: none 
node_mirror: http://npm.taobao.org/mirrors/node/ 
npm_mirror: https://npm.taobao.org/mirrors/npm/

                          # 3. nvm的常用命令

                          nvm version
nvm list
nvm ls                  #列出已安装实例（node/npm）
nvm ls available        #列出远程服务器上所有的可用版本 (linux/mac使用：nvm ls-remote)

nvm install 14.19       #每当我们安装了一个新版本 Node 后，全局环境会自动把这个新版本设置为默认
nvm install 16.14       #安装16.14

nvm use 16.14                       #在不同版本间切换 
nvm alias awesome-version 16.14     #给 16.14 这个版本号起了一个名字叫做 awesome-version
nvm use awesome-version
nvm unalias awesome-version         #取消别名

                          # 4. 指定项目的Node版本

                          可以通过创建项目目录中的 .nvmrc 文件来指定要使用的 Node 版本。 之后在项目目录中执行 nvm use 切换到对应的版本即可。

                          .nvmrc 文件内容只需要遵守上文提到的语义化版本规则即可。

                          另外还有个工具叫做 avn: https://github.com/wbyoung/avnopen in new window ，可以自动化这个过程。


                          在多环境中，npm该如何使用呢？每个版本的 Node 都会自带一个不同版本的 npm，可以用 npm -v 来查看 npm 的版本。

                          但问题来了，我们安装过的 npm 包，都要重新再装一次？幸运的是，我们有个办法来解决我们的问题，运行下面这个命令，可以从特定版本导入安装过的包到我们将要安装的新版本 Node：

                          nvm install 14.19 --reinstall-packages-from=12

                          # 二 前端依赖管理—npm

                          npm全名 node package manger ，是一款前端的包管理工具（作用参照Maven）

                          # 1. 安装和配置

                          npm是随同NodeJS一起安装的包管理工具

                          配置npm镜像地址：

                          npm -v                                                         #查看 npm 版本号

npm config list                                                #检查 npm 镜像地址
npm config set registry https://registry.npm.taobao.org        #配置 npm 淘宝镜像

# 安装使用cnpm
npm install -g cnpm --registry=https://registry.npm.taobao.org

                          部分npm组件调用node-gyp编译时只支持python2版本的问题

                          部分npm组件在安装时只支持python2版本，如果本地python环境变量是设置的python3，则会导致错误。 解决方法: 安装时执行，指定python版本

                          npm install --python=python2

                          或者直接修改npm的python版本设置

                          npm config set python python2

                          这样npm install时，如果需要调用node-gyp编译，会使用指定的python执行。

                          # 2. npm-查看模块

                          模块API查询：https://npm.taobao.org/open in new window

                          npm list -g                 # 查看全局模块
npm list                    # 查询已安装的本地模块，但前提是当前目录下有  node_modules  目录
 
npm list -g --depth=0       # 查看全局模块的一级目录
npm list --depth=1          # 查询本地模块的两级目录

                          --depth=0 表示查询模块，只显示第一层级的模块。这里的 0 如果换成 1 就是显示第一层级和第二层级的模块，依次类推

                          # 3. 模块的安装与卸载

                          局部安装和全局安装（-g）：

                          npm install express            #  .\node_modules
npm install express -g         #  C:\Program Files\nodejs\node_modules

                          查询全局模块及安装路径：

                          npm list -g --depth=0

                          安装指定版本的模块，同时安装多个模块：

                          npm install react-router@3.*           # 安装第三版中最新的包
npm install antd babel-plugin-import   # 同时安装多个模块

                          安装 package.json 文件中的包（当然前提是有 package.json 并且里面配置了相关包信息）, 或将安装信息记录到 package.json 文件中：

                          npm install                            #  安装 package.json 中的包，

#  --save  将安装的模块信息记录在  package.json  文件的dependencies属性下(生产)
npm install antd --save       

# - -save-dev 将模块信息记录到  package.json  文件的devDependencies属性下(开发)         
npm install antd --save-dev

                          良好的操作习惯是在安装模块时使用 --save 将安装了哪些模块都记录在 package.json 中，这样如果要查询可以直接去 package.json 中查看即可


                          模块的卸载：

                          npm uninstall lodash              # 只卸载lodash模块， package.json 中的记录仍然存在
npm uninstall lodash --save       # 卸载lodash模块的同时删除在 package.json 文件中的记录
npm uninstall lodash@3.*          # 卸载 lodash 模块 3.* 版本

                          应用场景举例

                          1. 在项目中先安装了 lodash 4.17.4 版本的模块；后来发现这个版本太高，存在一些新 bug 没有解决，不适合项目中使用；
                          2. 想要安装版本更加稳定的3.* 版本， 于是敲入指令 npm install lodash@3.*；
                          3. 注意: 虽然第二次安装了 3.* 版本，但是由于之前的 4.* 版本并没有卸载，此时在本地安装路径中可以发现有两个版本 lodash 的模块， 而 npm 默认在使用时会优先调用高版本的模块。
                          4. 这时将之前版本进行卸载，npm install lodash，会默认卸载高版本的模块。

                          # 4. npm-模块的更新

                          npm update lodash           # 更新到小版本号最新的那个版本，但不会更新到大版本号

npm update lodash --save    # 更新模块的同时将更新信息记录到  package.json  文件中

                          例：lodash 的版本号有： 3.9.1 、 3.10.1 、 4.13.1 ， 当前已经安装的 lodash 版本是 3.9.1 ， 此时执行下列命令：

                          npm update lodash@4.13.1 --save

                          此时由于对大版本号进行更新，结果没有任何响应，依然是 3.9.1 版本


                          下面只更新小版本号：

                          npm update lodash@3.10.* --save

                          此时并没有更改大版本号，更新结果为 lodash 的版本变成了 3.10.1 版本


                          # 5. npm版本规则

                          1. npm 发布包的版本指定规则：

                            使用 Npm 发布一个包的时候，往往要遵循 x.y.z 的规则，发布的第一个版本一般为 1.0.0。

                          • x 大版本号。引入新的变化，破坏向后兼容，x 值 +1 变成 2.0.0
                          • y 小版本号。增加一个新功能，且不影响已有功能，y 值 +1 变成 1.1.0
                          • z 补丁号。修改某个功能的 Bug 时，z 值 +1 变成 1.0.1

                          1. 关于向后兼容与向前兼容：

                            向前兼容和向后兼容可以用下面这个例子方便理解：

                          • 向后兼容：Windows 10要能运行为Windows 3.1开发的程序；
                          • 破坏向后兼容：新版本的系统不能运行老版本系统上的程序，可以理解为整容的跟以前完全不一样了；
                          • 向前兼容：Windows 3.1要能运行为Windows 10开发的程序。

                          1. package.json 中包版本 ~ 与 ^ 说明
                          • ~1.4.0表示：>=1.4.0 && < 1.5.0 说明：小版本不变，补丁号可以取最大值
                          • ^1.4.0表示：>=1.4.0 && < 2.0.0 说明：大版本号不变，小版本号可以取最大值

                          # 三 前端依赖管理—yarn

                          yarn是Facebook发布的一款npm包管理工具

                          • 安全: 在执行代码之前，Yarn会通过算法校验每个安装包的完整性

                          • 速度快: Yarn缓存了每个下载过的包，所以再次使用时无需重复下载

                          • 可靠: 项目中的依赖的版本会被锁定(yarn.lock)，不用担心协同开发过程中出现版本不一致的问题

                          # 1. NPM安装yarn

                          安装yarn的前提是需要先安装好npm，且环境变量都配置好了，才可以使用

                          下载并安装：

                          npm install yarn -g

                          安装完成之后检查版本：

                          yarn --version

                          yarn:无法加载文件

                          有时候安装完成后使用yarn命令，部分电脑会出现如下错误：

                          yarn : 无法加载文件 C:\Program Files\nodejs\yarn.ps1，因为在此系统上禁止运行脚本

                          解决办法：

                          powerShell 窗口上执行：

                          set-ExecutionPolicy RemoteSigned

                          可用如下命令查看执行策略是否设置成功：

                          get-ExecutionPolicy

                          # 2. yarn相关配置

                          yarn config set registry https://registry.npm.taobao.org -g
yarn config set sass_binary_site https://npm.taobao.org/mirrors/node-sass/ -g

                          检查配置：

                          yarn config get registry 
yarn config get sass_binary_site

                          # 3. Yarn常用命令

                          yarn init                      // 生成package.json文件
yarn install                   // 安装yarn.lock的所有依赖
yarn install --force           // 重新安装依赖

yarn remove moduleName         // 删除依赖
yarn add moduleName            // 安装某个依赖

yarn add moduleName --dev/-D   // 安装到开发环境
yarn run scriptName            // 执行package.json命名的脚本命令
