                      # IBM DB2 Express-C

                      # 一 DB2简介与安装

                      DB2是IBM出品的一系列关系型数据库管理系统，分别在不同的操作系统平台上服务

                      # 1. DB2版本及简介

                      DB2是IBM公司的关系型数据系统、适用于大型应用系统，如：某企业门户网站，商务智能系统。支持大多数主流操作系统，如：Windows，Linux，AIX

                      • 1.DB2 Express-C (社区免费版，提供了基本的数据库能力，启动速度快，适合企业做测试开发以及入门学习的人使用）
                      • 2.DB2 Express (是一种功能全面的支持Web 的客户机/服务器RDBMS，主要用于小型企业，收费)
                      • 3.DB2 Workgroup Server(DB2工作组版，适用于部门或者中型企业环境，也是全功能数据库)
                      • 4.DB2 Enterprise Server(企业服务器版，企业使用的最多，适合任何不同负载的工作环境，没有任何CUP，内存，数据库大小的限制)
                      • 5.DB2 Advanced Enterprise Server Edition(高级企业服务器版(包含企业服务器所有功能，额外还有一组工具组件)

                      # 2. Docker安装DB2

                      docker search db2                   # 查询可安装的DB2数据库

docker pull ibmcom/db2              # 下载db2镜像
docker images

# 使用DB2镜像启动一个容器
docker run -itd --name db2 --privileged=true --restart=always \
-p 50000:50000 \
-e LICENSE=accept \
-e DB2INST1_PASSWORD=123456 \
-e DBNAME=testdb  \
-v /docker/data/db2:/database ibmcom/db2

# 进入DB2容器
docker exec -it <container ID> /bin/bash

# 查看DB2数据库的版本及安装目录
db2level
db2 "select * from org"  # 查询表

                      DB2用户密码默认90天后过期，会导致无法连接数据库，自己的测试环境可以设置永不过期：

                      (以下操作在容器内部的root用户下进行)

                      more /etc/shadow             # 查看所有用户信息

chage -l db2inst1            # 查看db2inst1用户密码过期时间

chage -M 99999 db2inst1      # 设置为永不过期

                      连接DB2数据库:

                      jdbc:db2://192.168.42.10:50000/testdb        # url

                      从DB2 9开始，DB2唯一支持的通信协议是TCP/IP，IP端口号必须唯一，一般从50000开始，但不能超过65535。在配置连接前，需要确保两边的网络是通的，排除网络故障和防火墙等问题。


                      # 3. DB2权限控制

                      DB2数据库权限分为实例级权限（SYSADM、SYSCTRL、SYSMAINT、SYSMON）和DB2数据库级权限（DBAMD、LOAD）。DB2中用户所拥有的权限主要考虑三个方面：实例级、数据库级、数据库操作级别，查看命令如下：

                      db2 get authorizations       # 新版本已取消该命令，可通过syscat.dbauth查看

                      实例级权限: SYSADM 最高管理权限,通常供DBA使用 SYSCTRL 最高系统控制权限,有SYSMAINT 和SYSMON 的全部权限 SYSMAINT 次级系统 控制权限 SYSMON 最小的实例级管理权限

                      id db2inst1           # 显示真实有效的用户ID(UID)和组ID(GID)

                      DB2中的权限: https://www.cnblogs.com/HengChangXin/articles/4201320.htmlopen in new window


                      # 二 DB2体系结构


                      # 1. 实例和数据库

                      DB2的启动和关停是以instance为单位的。可以启动一个instance，或者关停一个instance。但不可以启动或者关停一个数据库或者一个schema。

                      db2start                   # 启动DB2实例
db2stop                    # 关闭DB2实例
db2stop force              # 强制停止实例

db2ilist                   # 查看所有实例
db2 get instance           # 查看当前实例

db2 list db directory      # 查看当前实例下已创建的数据库

                      查看实例配置参数：

                      db2 get dbm cfg            # 查看实例配置参数

                      实例参数只能通过update dbm cfg设置，绝大多数参数的更改都需要重启实例

                      db2 update dbm cfg using DFT_MON_STMT on

                      DATABASE数据库相关命令：

                      db2 get db cfg             # 查看数据库配置参数
db2 get db cfg for databaseName

db2 list active databases  # 列出活动的数据库

db2 connect to sample      # 连接到数据库

db2 terminate              # 断开数据库连接
db2 connect reset          # 断开连接
db2 force application all  # 强迫所有应用断开数据库连接(异步的)

db2 list tables            # 查看当前数据库中的所有表

                      连接远程数据库的方式：

                      db2 connect to <数据库名> user <用户名> using <密码>

                      # 2. 表空间和Schema

                      在列出表空间之前，必须要连接到该数据库，才可以查看该数据库下的表空间信息。 即需要先连接到数据库（以sample为例），再使用db2 list tablespaces命令查看表空间信息：

                      db2 connect to sample

db2 list tablespaces               # 查看当前数据库的表空间
db2 list tablespaces show detail   # 查看表空间详细信息

# 查看表空间的容器信息（及容器详细信息）
db2 list tablespace containers for 0 
db2 list tablespace containers for 0 show detail

                      Schema: 数据库中每一个表和索引等对象都有自己唯一的名字。但是，有的时候一个数据库可能被多个用户使用，如果用户A把自己的一个表叫做TEST，而用户B也想用同样的名字怎么办？这种情况下就引入了模式的概念。模式英文名为schema，简单地说，模式是对数据库对象的逻辑分组。

                      # 查看当前schema
db2 select current schema from sysibm.sysdummy1 

# 切换当前schema
db2 set current schema SCHEMA_Name

                      DB2模式支持两种创建方式，一种是通过create schema命令创建，这种叫显式创建；另外一种是在创建表时指定schema名，如果该schema以前没有创建，那么在创建表时会自动创建（前提是有权限），这种叫隐式创建。

                      db2 "create schema prod"      # 显示创建

# 在创建表时指定schema名（隐式创建）
db2 "create table test.t1 (product_name char(20))"

                      DB2 将schema下所有表的权限赋予一个用户

                      db2 "select 'db2 grant select on table DB2INST2.'|| tabname ||' to db2inst1' from syscat.tables where tabschema='DB2INST2'"

                      单独对某个Schema下的表进行授权

                      db2 grant select on table SchemaName.TableName to user userName

                      DB2中的用户和schema关系

                      熟悉Oracle的人都知道Oracle的schema和user是同一个概念；每一个Oracle user就是一个oracle schema。

                      Oracle和MySQL数据库的用户都是数据库内部的用户，由数据库来管理。

                      但DB2不一样，DB2没有数据库用户的概念，它依赖于操作系统用户，也就是操作系统上有这个用户，这个用户才可能连接到DB2数据库。

                      如果创建表的时候不指定schema、则使用当前用户的用户名作为默认的Schema

                      -- 查询当前存在的Schema
select SCHEMANAME,owner,CREATE_TIME from syscat.schemata

                      # 3. 表的相关操作

                      db2 list tables for all                    # 查看所有的表
db2 list tables for schema <schema_name>   # 查看某个模式下的表名

# 通过describe table命令查看表字段及数据类型
db2 describe table db2inst1.employee

                      创建表：(表空间需要事先规划并建好)

                      create table employee
(
    id       int not null primary key,
    name     char(10),
    age      int,
    edulevel int,
    resume   clob,
    comment  varchar(100)
)
    in data_ts            -- In指定了数据表空间
    index in index_ts     -- index in指定索引存储的表空间
    long in long_ts       -- long in指定大对象数据存储的表空间
